# Модульная архитектура проекта PROPELLINI

## Структура модулей

```
src/
├── modules/
│   ├── header/          # Навигация и шапка
│   │   ├── header.css
│   │   └── header.js
│   ├── hero/            # Главный блок
│   │   └── hero.css
│   ├── benefits/        # Преимущества
│   │   └── benefits.css
│   ├── calculator/      # Калькулятор стоимости
│   │   ├── calculator.css
│   │   └── calculator.js
│   ├── gallery/         # Галерея портфолио
│   │   └── gallery.css
│   ├── contacts/        # Контакты и форма
│   │   └── contacts.css
│   └── footer/          # Подвал сайта
│       └── footer.css
├── styles/
│   └── global.css      # Глобальные стили, переменные, reset
└── js/                  # Утилиты и вспомогательные скрипты
```

## Принципы модульности

1. **Изоляция**: Каждый модуль независим и содержит только свои стили/логику
2. **Переиспользование**: Глобальные стили в `global.css`
3. **Расширяемость**: Легко добавлять новые модули
4. **Поддерживаемость**: Изменения в одном модуле не ломают другие

## Порядок подключения

1. `global.css` - базовые стили, переменные
2. Модули по порядку: header → hero → benefits → calculator → gallery → contacts → footer
3. Legacy стили для совместимости
4. Mobile redesign в конце

## JavaScript модули

- `calculator.js` - основной класс калькулятора
- `header.js` - управление навигацией
- Остальные утилиты в `src/js/`

## Миграция

Старые файлы постепенно переносятся в модули. Legacy стили оставлены для обратной совместимости.
