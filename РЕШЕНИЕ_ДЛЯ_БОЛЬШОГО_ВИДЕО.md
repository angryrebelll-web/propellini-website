# Решение для загрузки большого видео файла

## Проблема
GitHub ограничивает размер файла до **100 MB**. Видео файл больше этого лимита.

## ✅ РЕШЕНИЕ: Используйте внешний хостинг для видео

### Рекомендация: Cloudflare Stream (БЕСПЛАТНО)

**Шаги:**

1. **Зарегистрируйтесь на Cloudflare Stream:**
   - Перейдите на https://dash.cloudflare.com/sign-up
   - Создайте бесплатный аккаунт

2. **Загрузите видео:**
   - Войдите в панель Cloudflare
   - Перейдите в раздел **Stream**
   - Нажмите **Upload Video**
   - Загрузите `teaser-desktop.mp4` и `teaser-mobile.mp4`

3. **Получите ссылки:**
   - После загрузки вы получите прямую ссылку на видео
   - Скопируйте эти ссылки

4. **Обновите HTML:**
   
   Замените в `index.html` строку 542:
   ```html
   <!-- БЫЛО: -->
   <source src="./public/images/video/teaser-desktop.mp4" type="video/mp4" />
   
   <!-- СТАЛО: -->
   <source src="ВАША_ССЫЛКА_ОТ_CLOUDFLARE" type="video/mp4" />
   ```

---

### Альтернатива 1: YouTube (Самый простой)

1. Загрузите видео на YouTube как "Не в списке"
2. Получите embed код
3. Замените `<video>` тег на YouTube iframe

**Пример:**
```html
<!-- Вместо video тега -->
<iframe 
  width="100%" 
  height="600" 
  src="https://www.youtube.com/embed/ВАШ_ID_ВИДЕО?autoplay=1&mute=1&loop=1&playlist=ВАШ_ID_ВИДЕО" 
  frameborder="0" 
  allow="autoplay; encrypted-media">
</iframe>
```

---

### Альтернатива 2: Git LFS (Если хотите использовать GitHub)

1. Установите Git LFS: https://git-lfs.github.com/
2. В папке проекта выполните:
```powershell
git lfs install
git lfs track "*.mp4"
git add .gitattributes
git add public/images/video/*.mp4
git commit -m "Add video with LFS"
```

⚠️ **Ограничение**: Бесплатно только 1 GB хранилища LFS

---

### Альтернатива 3: Удалить видео из репозитория

Если видео не критично для GitHub репозитория:

1. Добавьте в `.gitignore`:
```
public/images/video/*.mp4
```

2. Видео останется локально, но не будет загружаться на GitHub

3. Для production используйте внешний хостинг или загрузите видео на сервер отдельно

---
